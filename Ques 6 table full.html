  2. BACK END DEVELOPMENT

    3. Backend technology (or server software) coordinates the exchange of information between the frontend and the server running a webpage. It allows an application to communicate with the main server.

      You can group backend technology into three sub-layers:

    1. API layer

      2. Primany Use Case Key Features/ Benefits
      Receives interactions from the frontend and then communicates these to the storage layer
      • This layer acts as a bridge between the frontend and the backend.
      • allow businesses to easily integrate with external services and tools, expanding their capabilities.

    2. Storage layer

      3. Primany Use Case Key Features/ Benefits
      Manages and stores any application data
      • It will communicate with databases to write and read data
      • A notification layer will send notifications from the frontend to the backend and vice versa
      • This allows an application to communicate and trigger responses.

    3. Business logic layer

      4. Primany Use Case Key Features/ Benefits
      It is the main core of the backend
      • Backend developers configure processing logic in this layer
      • changing the response a certain API request delivers.
